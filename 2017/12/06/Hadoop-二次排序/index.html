Hadoop 二次排序

发表于 2017-12-06   |   分类于 教程   |     |   阅读次数

在 MapReduce 的输出结果中，每一个 Key 可能对应了不止一个 Value，我们需要让对应同一个 Key 的多个Value 按照一定的顺序排列，一种方法是在 Reduce 完成后对同一个 Key 对应的 Value 进行排序，但是对于 MapReduce 这样一个应用在大数据场景中的计算框架来说这种方法显然是不理想的，因为一个 Key 对应的 Value 可能有很多，以至于无法在单机内存中处理。理想的方法是在 MapReduce 的过程中就完成排序，由于MapReduce 的伸缩性，即使数据量很大也能高效地进行处理。另一方面，MapReduce确保每个 Reducer 的输入都是按 Key 排序的（《Hadoop 权威指南》6.4 shuffle和排序 P226），因此我们可以将 Key-Value 组合成一个中间键 newKey，在 Partition 和 Group 时考虑 Key，而在比较大小时考虑 newKey， 从而利用 Reducer 按键排序输入这一特性完成排序。
