字符串匹配之KMP算法

发表于 2017-12-13   |   分类于 算法   |   阅读次数

KMP 算法思路

关于字符串匹配，常规的思路是用原串中长度等于模式串的子串和模式串比较，如果二者相等那么就成功找到了一个匹配。假设原串长度为 $m$ ， 模式串长度为 $n$，那么我们需要做 $O(m)$ 次字符串比较，每次比较又需要做 $O(n)$ 次字符的比较，这样一来时间复杂度就为 $O(mn)$，并非一个理想的方法。
如果使用KMP算法，可以减少匹配中大量重复的匹配，将复杂度缩小到 $O(m + n)$。
KMP 的思路是：当遇到两个字符不匹配时（ $origin[p+1]$ 和 $pattern[q+1]$ ），在模式串中，我们从这个不匹配当位置往前看，找到一个最长的子串 $S_1$，这个子串必须满足 从模式串的起始位置开始也有一个一摸一样的子串 $S_0$，这样我们就能够把模式串整体右移，用 $S_0$ 去对齐上次匹配时原串中与 $S_1$对齐的位置。

这样我们可以直接令 $q = next(q)$ , 然后继续匹配 $origin[p+1]$ 和 $pattern[q+1]$，从而省去了常规思路中的回退和不必要的匹配。

next 数组

容易看出，我们要使用 KMP 算法时，需要预先计算出模式串中每个位置 $q$ 的 $next(q)$ , 得到一个$next$ 数组，然后每次不匹配时就去查这个数组，找到对应的值。根据上图可以看出：我们分别从 $pattern[0]$ 向后和从 $pattern[q]$向前去找最长的相同的字符串 $S_0$ 和 $S_1$ ， $S_0$ 的最后一位就是 $next(q)$ 。但是这样的话时间复杂度就正比与 $q$，而求解整个 $next$ 数组的时间复杂度就是 $O(n^2)$ , 显然不是一个满意的结果。但是如果你用该方法去求解一个模式串的 $next$ 数组后你就会发现起始 $next(q)$ 和 $next(q-1)$ 是相关的。如果 $pattern[q] = pattern[next(q-1)+1]$ ,那么 $next(q) = next(q-1)+1 $ ,并且这种关系是可以一直递推下去的，这样我们就能近似在 $O(n)$ 的时间复杂度内求出 $next$ 数组。
接下来我们以模式 bababb 为例求解它的 $next$ 数组：
1.$\quad$首先将 $next(0)$ 置为 -1

0 1 2 3 4 5
b a b a b b
-1 ? ? ? ? ?

2.$\quad pattern[1] = a \ne pattern[next(1-1) + 1] = b$ ，这里 $ next(1-1) = -1$ 而 $next(-1)$ 不存在所以无法继续递推，故 $next(1) = -1$

0 1 2 3 4 5
b a b a b b
-1 -1 ? ? ? ?

3.$\quad (pattern[2] = b) = (pattern[next(2-1) + 1] = pattern[0] = b)$ 故 $next(2) = 0$

0 1 2 3 4 5
b a b a b b
-1 -1 0 ? ? ?

4.$\quad (pattern[3] = a) = (pattern[next(3-1) + 1] = pattern[1] = a)$ 故 $next(3) = 1$

0 1 2 3 4 5
b a b a b b
-1 -1 0 1 ? ?

5.$\quad(pattern[4] = b) = (pattern[next(4-1) + 1] = pattern[2] = b)$ 故 $next(3) = 2$

0 1 2 3 4 5
b a b a b b
-1 -1 0 1 2 ?

6.$\quad (pattern[5] = b) \ne (pattern[next(5-1) + 1] = pattern[3] = a)$
接着 $pattern[next(next(5-1)) + 1] = a$ 也不等于 $pattern[5]$ 这样一直到 $ next(-1)$ 也没找到相等的字符，但注意到 $pattern[0] = b = pattern[5]$ , 所以 $next(5) = 0$ (和 $next(1)$ 不一样 )

0 1 2 3 4 5
b a b a b b
-1 -1 0 1 2 0
